Theory Reference

C

Dennis Ritchie, Bell Labs · 1972

Statically typed, structured, procedural language. The functional foundation for modern operating systems, combining low-level memory access with system architecture performance.

  • Compilation pipeline: Preprocess → Compile → Assemble → Link into a single binary.
  • Explicit pointers hold hardware memory addresses; use malloc/calloc/free for the heap.
  • Platform-dependent primitive sizes (int is 4 bytes on most 64-bit systems).

Python

Guido van Rossum · 1991

High-level, dynamically typed, interpreted language. Emphasises runtime accessibility and clean syntax across OOP, procedural, and functional styles.

  • Source (.py) is parsed to bytecode (.pyc) and executed on the CPython VM.
  • Variables are dynamic references; memory is reclaimed by reference counting + GC.
  • First-class collections: Lists, Tuples, Dicts, Sets.

Java

James Gosling, Sun Microsystems · 1995

Class-based, strictly typed, concurrent OOP language. 'Write Once, Run Anywhere' via the JVM isolates code from hardware differences.

  • javac emits bytecode (.class); the JVM interprets, with JIT for hot paths.
  • Four pillars: Encapsulation, Inheritance, Polymorphism, Abstraction.
  • No raw pointers; the Garbage Collector reclaims managed heap objects.
